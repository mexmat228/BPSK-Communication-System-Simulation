# Моделирование BPSK системы связи (текстовая версия)

## Описание программы

Данная программа реализует полное моделирование цифровой системы связи с использованием **BPSK модуляции** (Binary Phase Shift Keying — бинарная фазовая манипуляция) в канале с **аддитивным белым гауссовским шумом (AWGN)**. Это полностью текстовая версия, которая работает в любом окружении Octave/MATLAB, даже при отсутствии поддержки графики.

## Принцип работы

Программа последовательно выполняет следующие этапы:

1. **Генерация случайных битов** — создается последовательность из 100 000 случайных нулей и единиц
2. **BPSK модуляция** — биты преобразуются в символы: `0 → -1`, `1 → +1`
3. **Моделирование канала связи** — к сигналу добавляется гауссовский шум с заданным отношением сигнал/шум (SNR)
4. **Демодуляция** — принятые сигналы декодируются обратно в биты по правилу: `>0 → 1`, `<0 → 0`
5. **Расчет ошибок** — сравниваются переданные и принятые биты, вычисляется BER (Bit Error Rate)
6. **Теоретическое сравнение** — экспериментальные результаты сопоставляются с теоретической формулой `BER = 0.5 * erfc(√(SNR))`

## Параметры моделирования

| Параметр | Значение | Описание |
|----------|----------|----------|
| `numBits` | 100 000 | Количество передаваемых битов |
| `SNR_dB` | 0, 2, 4, 6, 8, 10, 12 дБ | Уровни отношения сигнал/шум |

## Что выводит программа

Программа генерирует подробный текстовый отчет, который включает:

### 1. Таблицу результатов
Сравнение экспериментальной и теоретической BER для каждого уровня SNR с указанием количества ошибок.

### 2. ASCII визуализацию BER
Текстовые полоски, показывающие значение BER в логарифмическом масштабе:
- `[====]` — экспериментальные значения
- `{====}` — теоретические значения

### 3. Визуализацию ошибок
Столбчатую диаграмму из символов `█`, где каждый символ представляет определенное количество ошибок.

### 4. Анализ улучшения производительности
Расчет снижения количества ошибок при увеличении SNR (в процентах и разах).

### 5. Анализ целевой BER
Определение минимального SNR, необходимого для достижения BER ≤ 10⁻³ (одна ошибка на 1000 бит).

### 6. Проверку точности
Сравнение экспериментальных и теоретических результатов с оценкой качества совпадения.

### 7. Сводную статистику
Общую информацию о симуляции, лучшей и худшей производительности.

### 8. Выводы
Ключевые результаты моделирования и практические рекомендации.

## Результаты моделирования

Типичные результаты работы программы:

| SNR (дБ) | BER (Эксперимент) | BER (Теория) | Ошибки |
|----------|-------------------|--------------|--------|
| 0 | ~1.25×10⁻¹ | ~1.29×10⁻¹ | ~12 500 |
| 6 | ~4.12×10⁻³ | ~4.12×10⁻³ | ~412 |
| 12 | ~1×10⁻⁵ | ~1×10⁻⁷ | 0-1 |

## Ключевые выводы

- Увеличение SNR с 0 дБ до 12 дБ снижает количество ошибок более чем на 99%
- Экспериментальные результаты хорошо совпадают с теоретическими
- Для надежной связи (BER < 10⁻³) требуется SNR ≥ 8 дБ
- Каждые 2 дБ увеличения SNR уменьшают количество ошибок в 3-5 раз

## Применение

Результаты моделирования применимы для:
- Оценки качества каналов связи (Wi-Fi, Bluetooth, сотовая связь)
- Выбора мощности передатчика для обеспечения надежной связи
- Понимания фундаментальных принципов цифровой коммуникации
- Обучения основам теории связи и обработки сигналов

## Системные требования

- **GNU Octave** (версия 3.8 или выше) или **MATLAB**
- Отсутствие требований к графическим пакетам (работает даже в онлайн-компиляторах)
